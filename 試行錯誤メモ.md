# デプロイメント・インシデントレポート：Astro v6 SSRアプリのCloudflareへのデプロイ

## 1. プロジェクトの前提条件と初期状態

- **フレームワーク**: Astro v6 (`@astrojs/cloudflare` v13)
- **バックエンド**: Supabase, Google Calendar API
- **目的**: ローカル環境で正常動作するSSRアプリケーションを、Git push経由でCloudflareへ自動デプロイする。
- **初期アプローチ**: Cloudflare Pagesへのデプロイを試行。

---

## 2. 発生した問題と対応策の時系列記録

### フェーズ1：Cloudflare PagesでのSSR 404エラー

- **事象**: Git pushによるビルド・デプロイは成功するが、Webアプリへアクセスすると `404 Not Found` が返却される。
- **原因**: `@astrojs/cloudflare` アダプターの最新バージョンは、標準で「Cloudflare Pages」ではなく「Cloudflare Workers」向けのSSR出力を生成する仕様に変更されている。Pages環境にデプロイしようとしたため、サーバー処理のスクリプト（`_worker.js`等）が正しく認識・実行されず、静的ファイル配信のみが行われていた。
- **対応策**: デプロイメントのターゲットを「Cloudflare Pages」から**「Cloudflare Workers」**へ変更した。Cloudflareダッシュボードにて新規のWorkersプロジェクトを作成し、GitHubリポジトリと連携させた。

### フェーズ2：ビルド時の互換性日付 (Compatibility Date) エラー

- **事象**: Workers環境でのビルド中に `This Worker requires compatibility date "YYYY-MM-DD", but the newest date supported by this server binary is "YYYY-MM-DD"` というエラーでクラッシュする。
- **原因**: `wrangler.json` に `compatibility_date` の指定がない場合、ビルドツール（Miniflare）は自動的に現在の日付を使用するが、インストールされているツールチェーンのサポート上限日付を超過したためパニックを引き起こした。
- **対応策**: プロジェクトルートの `wrangler.json` に明示的な互換性日付を設定した。
  ```json
  {
    "name": "equipment-reservation",
    "compatibility_date": "2026-04-18",
    "compatibility_flags": ["nodejs_compat"]
  }
  ```

### フェーズ3：エントリポイント検証エラー (Chicken and Egg問題)

- **事象**: デプロイステップにおいて `Missing entry-point to Worker script` または `The provided Wrangler config main field doesn't point to an existing file` が発生し、デプロイが完了しない。
- **原因**: Wranglerがアップロード対象を特定するためには `wrangler.json` に `main` および `assets` プロパティの指定が必要である。しかし、指定を追記すると、Astro (Vite) がビルドを**開始する前**に設定ファイルを評価し、対象ファイル（ビルド前なので未生成）が存在しないと判定してビルド自体を強制終了させてしまう。
- **対応策**: ビルド中は `wrangler.json` に対象パスを記述せず（Viteの検証を回避）、Astroのビルドが完了した**直後**に、Node.jsスクリプトを用いて動的に `wrangler.json` へパス設定を追記する機構を採用した。

  _修正された `package.json` のビルドスクリプト:_

  ```json
  "scripts": {
    "build": "astro build && node -e \"const fs=require('fs'); const c=JSON.parse(fs.readFileSync('wrangler.json')); c.main=fs.existsSync('./dist/_worker.js/index.js')?'./dist/_worker.js/index.js':'./dist/server/entry.mjs'; c.assets={binding:'ASSETS',directory:'./dist'}; fs.writeFileSync('wrangler.json', JSON.stringify(c, null, 2));\""
  }
  ```

---

## 3. 最終的な構成状態（正常動作環境）

- **ホスティング**: Cloudflare Workers
- **Wrangler設定 (`wrangler.json` の初期状態)**:
  `name`, `compatibility_date`, `compatibility_flags` (`nodejs_compat`) のみを記述。
- **ビルドパイプライン**:
  1. `astro build` を実行（SSRファイルを生成）。
  2. ビルド後スクリプトにより `wrangler.json` に `main` と `assets` を自動追記。
  3. Wranglerが更新された設定を用いてデプロイを実行。
